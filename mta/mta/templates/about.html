About

This is a description of how and why this build exists. Please keep in mind that some of this might be a bit of overkill, but I am attempting to use this as a site for my portfolio.

The direction I am taking this site

This started out as a code challenge for an interview, but I am turning it into a full production site and portfolio of sorts to show what I am capable of.
I will be moving forward with the idea that this site will eventually be used throughout the world with millions of users, therefore I will be re-designing it as I go.
This will provide a living example of how a site might evolve over time (in Github) for people to follow and use as they grow and learn also.

Current goals

My current mind set has me working on:
  • A key:value caching layer that will store all of the provider-service-line data for quick recall; I am thinking of using Redis. The current process is a one off scenario
  • Ansible build

Future goals

  • I will be creating a seperate server for RabbitMQ and Celery in order to kick off the scheduled mining of status information
  • Load Balancer (EC2 initially)
  • Geospacial driven content, including ads, alerts, and maps to list a few
  • Harden with iptables, users, groups, permissions
  • Domain and SSL
  • Flexible storage, currently using the default for django (sqlite3, not meant for production); I am thinking Casandra (will go into availability later).
  • S3 for local centralized media
  • Centralized logging
  • Vagrant build
  • Docker
  • Create RDS instance, perhaps MySQL for a live backup
  • API perhaps in bottle
  • maybe Node.js for frontend
